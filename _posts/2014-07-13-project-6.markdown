#### ▪️프로젝트 개요
<p style="font-size: 0.95em; line-height: 1.6; color: #555;">

  <br>

#### ▫️담당 역할

1. **이슈 & AI 서버 파이프라인**: 이슈 CRUD 개발 및 AI 분석 서버 연동을 위한 비동기 파이프라인 구축

3. **실시간 통신 아키텍처**: Redis Pub/Sub 및 SSE를 활용한 실시간 알림 핸들링 전담

5. **핵심 도메인 로직**: 회의실 생성, 상태 관리 등 회의 관련 주요 비즈니스 로직 설계

7. **보안 및 인증**: Spring Security 기반의 안정적인 인증 프로세스 구현

---

#### ⚡주요 구현 기능 및 기술적 도전

**1. Redis & SSE를 활용한 비동기 분석 시스템 구축**

1. **문제**: AI의 이슈 우선순위 분석이 5~10초가 소요되어 사용자 화면 멈춤 및 타임아웃 위험 발생

2. **해결**: 
   * 분석 요청 즉시 202 응답을 반환하는 비동기 구조 설계
   * Redis Pub/Sub을 도입해 AI 서버의 분석 완료 메시지를 수신하도록 구현
   * SSE를 통해 클라이언트에게 실시간 분석 완료 알림 및 데이터 푸시

3. **결과**: AI 분석 중에도 자유로운 서비스 이용이 가능해지며 사용자 경험 극대화

<br>

**2. No-Offset 기반의 대용량 데이터 조회 최적화**

1. **문제**: 누적되는 방대한 채팅 및 이슈 조회 시 Offset 기반 페이징의 성능 저하 우려

2. **해결**: QueryDSL을 활용한 No-Offset(Cursor 기반) 방식의 무한 스크롤 API 구현

3. **결과**: 인덱스 스캔 효율을 높여 데이터가 늘어나도 일정한 조회 성능 유지 및 서버 자원 점유 시간 감소

<br>

**3. 도메인 중심 설계(DDD) 및 데이터 정합성 확보**

<div align="center">
  <img src="/img/portfolio/erd.png" width="70%" alt="ERD 구조도">
  <br>
  <small>[그림 1] DIDIT 서비스 논리 ERD 구조도</small>
</div>

1. **내용**:
   * GitHub API 연동을 통해 커밋/이슈 상태를 실시간으로 동기화하고, 이를 AI 서버로 전달하여 우선순위를 자동 산출하는 파이프라인 구축
   * JPA 연관관계 최적화: 지연 로딩 및 Fetch Join을 통해 N+1 문제를 방지하며 회의실-이슈-유저 간의 복잡한 흐름 정리

2. **성과**: 복잡한 비즈니스 로직을 도메인 계층으로 응집시켜 코드 유지보수성 향상

<br>

**4. 보안 및 인증**

1. **구현**: Spring Security를 적용하여 보안 취약점을 방어하고, 세션/토큰 관리를 통한 안정적인 로그인/로그아웃 프로세스 구축
