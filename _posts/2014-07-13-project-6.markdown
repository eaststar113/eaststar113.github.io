### 🚀 프로젝트 개요
개발자들이 겪는 **컨텍스트 스위칭(Context Switching)** 문제를 해결하기 위해, 회의록 자동 생성 및 깃허브 이슈 자동 분석 기능을 제공하는 서비스입니다.

### 🛠 담당 역할 (Individual Contribution)
* **이슈 & AI 서버 파이프라인**: 이슈 CRUD 개발 및 AI 분석 서버 연동을 위한 비동기 파이프라인 구축
* **실시간 통신 아키텍처**: Redis Pub/Sub 및 SSE를 활용한 실시간 알림 핸들링 전담
* **핵심 도메인 로직**: 회의실 생성, 상태 관리 등 회의 관련 주요 비즈니스 로직 설계
* **보안 및 인증**: Spring Security 기반의 안정적인 인증 프로세스 구현

### 💡 기술적 도전 및 해결 (Key Achievements)

#### 1️⃣ Redis & SSE를 활용한 비동기 분석 시스템 구축
* **문제**: AI의 이슈 우선순위 분석이 5~10초가 소요되어 사용자 화면 멈춤 및 타임아웃 위험 발생
* **해결**: 
    * 분석 요청 즉시 202 응답을 반환하는 **비동기 구조** 설계
    * **Redis Pub/Sub**을 도입해 AI 서버의 분석 완료 메시지를 수신하도록 구현
    * **SSE(Server-Sent Events)**를 통해 클라이언트에게 실시간 분석 완료 알림 및 데이터 푸시
    * 비동기 요청 타임아웃을 30분으로 최적화하여 긴 분석 작업의 안정성 확보
* **결과**: AI 분석 중에도 자유로운 서비스 이용이 가능해지며 **사용자 경험(UX) 극대화**

#### 2️⃣ No-Offset 기반의 대용량 데이터 조회 최적화
* **문제**: 누적되는 방대한 채팅 및 이슈 조회 시 Offset 기반 페이징의 성능 저하 우려
* **해결**: **QueryDSL을 활용한 No-Offset(Cursor 기반)** 방식의 무한 스크롤 API 구현
* **결과**: 인덱스 스캔 효율을 높여 데이터가 늘어나도 일정한 조회 성능 유지

#### 3️⃣ 도메인 중심 설계(DDD) 및 데이터 정합성 확보
* **내용**: 깃허브 이슈와 일반 이슈를 통합 관리하는 유연한 ERD 설계
* **성과**: JPA 연관관계 매핑 최적화를 통해 회의실-이슈-유저 간의 복잡한 데이터 흐름을 명확하게 정의

### 🛠 기술 스택 (Tech Stack)
* **Language/Framework**: Java 21, Spring Boot 4.0.1, Spring Security
* **Persistence**: Spring Data JPA, QueryDSL, MySQL 8.0
* **Messaging/Cache**: Redis 7.2 (Alpine)
* **Media**: OpenVidu 2.32.1
* **DevOps**: Docker, Jenkins, Nginx (Reverse Proxy & SPA Routing)
